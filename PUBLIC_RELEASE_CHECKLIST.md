# Public Release Checklist

This checklist should be completed before the repository is switched from private to public and before broad outreach to reviewers or potential collaborators.

## Repository Basics

- [ ] `README.md` explains status, scope, limitations, and collaboration need.
- [ ] `LICENSE.md` exists and distinguishes documentation, code, data, and sensitive material.
- [ ] `CITATION.cff` exists.
- [ ] `CONTRIBUTING.md` exists.
- [ ] `CODE_OF_CONDUCT.md` exists.
- [ ] AI-use transparency is documented.
- [ ] Public-readiness lint script runs successfully or known exceptions are documented.

## Scientific Readiness

- [ ] Research gap is presented as provisional, not proven.
- [ ] Existing facilities are treated as comparators, not dismissed.
- [ ] Literature review separates peer-reviewed sources, institutional sources, and open gaps.
- [ ] Central references have verified metadata.
- [ ] Claims not yet supported are explicitly marked as unsupported or hypothetical.

## Engineering Readiness

- [ ] Physics equations are readable and explained.
- [ ] Calculation scripts are linked from relevant documents.
- [ ] Railway, maglev, rotating, payload-only, and no-build options are compared neutrally.
- [ ] No architecture is presented as selected.
- [ ] Stop/go decision points are visible.

## Safety, Ethics, and Governance

- [ ] Human studies are clearly deferred.
- [ ] Animal studies are clearly deferred and require 3R justification.
- [ ] Biological payloads require biosafety and containment review.
- [ ] Human data are not open by default.
- [ ] Medical, clinical, athlete-performance, and safety claims are cautious.

## Readability

- [ ] No reader-facing raw LaTeX delimiters remain.
- [ ] Formulas use readable notation and symbol explanations.
- [ ] Internal links are current.
- [ ] Documents use consistent headers.
- [ ] Markdown documents have the project footer unless deliberately exempted.

## Outreach Readiness

- [ ] External-review request template is prepared.
- [ ] The ask is critique, not endorsement.
- [ ] The first milestone is narrow and testable.
- [ ] Reasons to stop or narrow the project are welcomed.
